## 一、概述

### 1.1 文档目的

本文档定义 Handsome Agent 项目的测试计划，包括测试范围、策略、资源、进度安排和风险评估。

### 1.2 项目背景

Handsome Agent 是一个企业级 AI Agent 系统，核心功能包括：
- LLM 驱动的决策引擎
- ReAct 循环执行引擎
- 50+ 预置工具
- 技能系统
- Curator 自我进化机制
- Rail 拦截器

### 1.3 测试目标

| 目标 | 说明 |
|------|------|
| **功能覆盖** | 所有功能需求都有对应的测试用例 |
| **代码质量** | 核心模块覆盖率 ≥ 80% |
| **缺陷控制** | 严重缺陷遗留率 < 1% |
| **回归保证** | 每次发布前完成回归测试 |

## 三、测试策略

### 3.1 测试分层

```
┌─────────────────────────────────────────┐
│              测试金字塔                   │
└─────────────────────────────────────────┘

                    ▲
                   ╱ ╲
                  ╱   ╲
                 ╱     ╲
                ╱  E2E  ╲           <- 少量，高价值
               ╱─────────╲
              ╱           ╲
             ╱  Integration ╲        <- 中等数量
            ╱───────────────╲
           ╱                 ╲
          ╱    Unit Tests     ╲      <- 大量，快速反馈
         ╱─────────────────────╲
```

| 层级 | 占比 | 运行频率 | 测试对象 |
|------|------|----------|----------|
| 单元测试 | 70% | 每次提交 | 核心模块、工具函数 |
| 集成测试 | 20% | 每天 | 模块间协作、API |
| E2E 测试 | 10% | 发布前 | 核心用户流程 |

### 3.2 测试方法

| 方法 | 说明 | 适用场景 |
|------|------|----------|
| **黑盒测试** | 基于需求规格，不了解内部实现 | 功能验证 |
| **白盒测试** | 基于代码结构设计测试用例 | 单元测试、覆盖率 |
| **边界测试** | 测试边界值和极端情况 | 输入验证 |
| **错误推断** | 基于经验预测可能错误 | 异常处理 |
| **场景测试** | 模拟真实用户操作 | E2E 测试 |

### 3.3 测试自动化策略

| 场景 | 自动化程度 | 工具 |
|------|-----------|------|
| 单元测试 | 100% | pytest + pytest-cov |
| API 测试 | 90% | pytest + httpx |
| CLI 测试 | 80% | pytest + click |
| 集成测试 | 80% | pytest + pytest-asyncio |
| 性能测试 | 按需 | pytest-benchmark |

## 六、风险评估

### 6.1 测试风险

| 风险 | 影响 | 概率 | 应对措施 |
|------|------|------|----------|
| LLM API 不稳定 | 高 | 中 | 使用 Mock Server 替代 |
| 测试环境资源不足 | 中 | 低 | 提前申请资源 |
| 测试数据准备延迟 | 中 | 中 | 并行准备测试数据 |
| 自动化框架不稳定 | 中 | 中 | 预留缓冲时间 |
| 需求变更 | 高 | 高 | 敏捷响应，及时更新用例 |

### 6.2 缺陷管理

| 严重级别 | 定义 | 响应时间 | 修复时间 |
|----------|------|----------|----------|
| **Critical** | 系统崩溃、数据丢失 | 1 小时 | 4 小时 |
| **High** | 核心功能不可用 | 4 小时 | 24 小时 |
| **Medium** | 功能缺陷，影响部分用户 | 24 小时 | 72 小时 |
| **Low** | UI 问题、文档错误 | 72 小时 | 下一版本 |

## 七、测试环境

### 7.1 测试环境配置

```yaml
# tests/pytest.ini
[pytest]
testpaths = tests
python_files = test_*.py
python_classes = Test*
python_functions = test_*
asyncio_mode = auto
addopts = -v --tb=short

# 环境配置
[pytest:env]
HANDSOME_LLM_PROVIDER = mock
HANDSOME_DATA_DIR = /tmp/test_data
```

### 7.2 测试依赖

```txt
# tests/requirements.txt
pytest>=8.0.0
pytest-cov>=4.0.0
pytest-asyncio>=0.23.0
pytest-mock>=3.12.0
httpx>=0.27.0
```

### 7.3 CI/CD 集成

```yaml
# .github/workflows/test.yml
name: Test

on:
  push:
    branches: [main, develop]
  pull_request:
    branches: [main]

jobs:
  unit-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: "3.11"
      - run: pip install -e ".[test]"
      - run: pytest tests/unit/ -v --cov=. --cov-report=term-missing
      - uses: codecov/codecov-action@v3

  integration-test:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - run: pip install -e ".[test]"
      - run: pytest tests/integration/ -v
```
