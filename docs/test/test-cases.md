## 一、概述

### 1.1 用例编写规范

```markdown
## 用例编号: 模块-功能-序号
**用例标题**: 简明描述
**前置条件**: 执行前必须满足的条件
**测试步骤**: 
1. 步骤 1
2. 步骤 2
**预期结果**: 期望的系统行为
```

### 1.2 用例状态说明

| 状态 | 说明 |
|------|------|
| **Draft** | 草稿，待评审 |
| **Ready** | 已评审，待执行 |
| **Pass** | 执行通过 |
| **Fail** | 执行失败 |
| **Blocked** | 阻塞，无法执行 |
